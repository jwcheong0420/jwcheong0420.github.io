proc file system

proc file systemмқҙлһҖ

  • Unix кі„м—ҙ OSм—җм„ң н”„лЎңм„ёмҠӨмҷҖ лӢӨлҘё мӢңмҠӨн…ң м •ліҙлҘј кі„мёөм Ғмқё нҢҢмқј кө¬мЎ° к°ҷмқҖ нҳ•мӢқмңјлЎң ліҙм—¬мЈјлҠ” pseudo filesystem
  • л¶ҖнҠё нғҖмһ„м—җ /procмқҙлқјлҠ” л§ҲмҡҙнҠё нҸ¬мқёнҠём—җ л§Өн•‘
  • м»Өл„җм—җм„ң лӮҙл¶Җ лҚ°мқҙн„° кө¬мЎ°мІҙм—җ лҢҖн•ң мқён„°нҺҳмқҙмҠӨмІҳлҹј н–үлҸҷ
    • м»Өл„җ мҳҒм—ӯкіј мӮ¬мҡ©мһҗ мҳҒм—ӯ мӮ¬мқҙмқҳ нҶөмӢ м—җ лҢҖн•ң л°©мӢқ м ңкіө
  • лҹ°нғҖмһ„(sysctl) мӢңм—җ нҠ№м •н•ң м»Өл„җ нҢҢлқјлҜён„°лҘј л°”кҫёкі  мӢңмҠӨн…ңм—җ лҢҖн•ң м •ліҙлҘј м–»лҠ”лҚ° мӮ¬мҡ©лҗ  мҲҳ мһҲмқҢ

/proc н•ҳмң„ кө¬мЎ°

  • /proc/{PID}
    • (м»Өл„җ н”„лЎңм„ёмҠӨлҘј нҸ¬н•Ён•ҳлҠ”) мӢӨн–ү мӨ‘мқё н”„лЎңм„ёмҠӨ лі„лЎң л””л үнҶ лҰ¬к°Җ мЎҙмһ¬
  • /proc/crypto
    • мӮ¬мҡ© к°ҖлҠҘн•ң м•”нҳёнҷ” лӘЁл“Ҳл“Өм—җ лҢҖн•ң лӘ©лЎқ
  • /proc/devices
    • мһҘм№ҳ IDм—җ мқҳн•ҙ м •л ¬лҗң мәҗлҰӯн„°мҷҖ лё”лЎқ мһҘм№ҳ лҝҗ м•„лӢҲлқј /dev мқҙлҰ„м—җ лҢҖн•ң лӘ©лЎқ
  • /proc/filesystems
    • лҰ¬мҠӨнҢ… мӢңмқҳ м»Өл„җм—җ мқҳн•ҙ м§ҖмӣҗлҗҳлҠ” нҢҢмқј мӢңмҠӨн…ңл“Өмқҳ лӘ©лЎқ
  • /proc/sys
    • лҸҷм ҒмңјлЎң м„Өм • к°ҖлҠҘн•ң м»Өл„җ мҳөм…ҳл“Өм—җ лҢҖн•ң мқён„°нҺҳмқҙмҠӨ
  • (мғқлһө)

мҷң proc file systemмқ„ мӮ¬мҡ©н•ҳлҠ”к°Җ?

  • нҢҢмқј мӢңмҠӨн…ң мҳӨлІ„ н—Өл“ңлҘј мӨ„мқј мҲҳ мһҲмқҢ
    • мқјл°ҳм Ғмқё нҢҢмқјмӢңмҠӨн…ңмқҖ мҳӨлІ„н—Өл“ңк°Җ л§ҺлӢӨ
      • к°Ғ нҢҢмқјмқҳ inode, superblocksмҷҖ к°ҷмқҖ к°қмІҙлҘј кҙҖлҰ¬н•ҙм•ј н•Ё
      • мқҙлҹ° м •ліҙлҘј н•„мҡ”н•  л•Ңл§ҲлӢӨ OSм—җ мҡ”мІӯн•ҙм•ј н•Ё
      • мқҙлҹ° м •ліҙл“ӨмқҖ м„ңлЎң м–ҙкёӢлӮ  мҲҳлҸ„ мһҲкі , лӢЁнҺёнҷ” нҳ„мғҒмқҙ л°ңмғқн•  мҲҳлҸ„ мһҲмқҢ
    • procfsмқҖ лҰ¬лҲ…мҠӨ м»Өл„җм—җм„ң м§Ғм ‘ нҢҢмқјмӢңмҠӨн…ңмқ„ кҙҖлҰ¬н•ҳлҠ” л°©лІ•мқ„ мұ„нғқ
    • procfsмқҖ м»Өл„җл©”лӘЁлҰ¬м—җм„ң лҸҢм•„к°ҖлҠ” к°ҖмғҒ нҢҢмқј мӢңмҠӨн…ң
  • л¬јлҰ¬м Ғмқё нҢҢмқјмӢңмҠӨн…ң мһҘм№ҳлҘј н•„мҡ”лЎң н•ҳм§Җ м•ҠмқҢ
    • м»Өл„җ л©”лӘЁлҰ¬м—җм„ң мң м§Җн•ҳлҠ” нҢҢмқј мӢңмҠӨн…ң
  • мөңм Ғнҷ”лҗң нҢҢмқј мһ‘м—… мҲҳн–ү
    • proc н•ҳмң„ нҢҢмқјл“ӨмқҖ кі м •м Ғмқҙкі  мІҳлҰ¬н•ҙм•јн•  лҚ°мқҙн„° м–‘мқҙ м ҒмңјлҜҖлЎң, open/read/write/close л“ұмқ„ мӮ¬мҡ©н•ҳм—¬ лҚ°мқҙн„°лҘј лӢӨлЈЁлҠ” кІғмқҙ л№„нҡЁмңЁм Ғ

proc н”„лЎңк·ёлһҳл°Қ

userмҷҖ лҚ°мқҙн„° көҗнҷҳлІ•
  • procfsмқҳ лҚ°мқҙн„°лҠ” нҢҢмқјм—җ м ҖмһҘлҗҳлҠ” кІғмқҙ м•„лӢҲлқј м»Өл„җ л©”лӘЁлҰ¬м—җ м ҖмһҘн•ҳл©°, м»Өл„җ л©”лӘЁлҰ¬лҠ” мң м Җл ҲлІЁм—җм„ң м§Ғм ‘ м ‘к·ј л¶Ҳк°ҖлҠҘ
    • userк°Җ cat(нҳ№мқҖ read н•ЁмҲҳ)лҘј нҶөн•ҙ нҢҢмқјмқҳ лӮҙмҡ©мқ„ мқҪмңјл Өн•ҳл©ҙ, м»Өл„җм—җм„ң лҚ°мқҙн„°лҘј мң м Җм—җкІҢ мқјм •н•ң нҸ¬л§·мңјлЎң лҝҢл ӨмӨҢ
    • userк°Җ м–ҙл–Ө лӮҙмҡ©мқ„ proc нҢҢмқјм—җ м“°кІҢлҗҳл©ҙ лҚ°мқҙн„°лҘј л°ӣм•„л“Өмқҙкі  к°Җкіөн•ҙм„ң м»Өл„җ л©”лӘЁлҰ¬м—җ лЎңл“ң
  • kernelмҷҖ user мӮ¬мқҙм—җ лҚ°мқҙн„°лҘј м „лӢ¬н•ҙмЈјлҠ” callback н•ЁмҲҳлҘј л“ұлЎқмӢңмјңм„ң мӮ¬мҡ©

  • лҸҷмһ‘

    • sysctl -w {param}=вҖқ{value}вҖқ

      • мһ¬л¶ҖнҢ… мӢң мң м§Җлҗҳм§Җ м•ҠмқҢ

      • file_pathлЎңл¶Җн„° sysctl м„Өм •мқ„ лЎңл“ң
      • file_pathк°Җ мЈјм–ҙм§Җм§Җ м•Ҡмңјл©ҙ /etc/sysctl.confмқ„ лЎңл“ң
